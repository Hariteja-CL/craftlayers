\#\# CraftLayers Portfolio Website: Mandatory Constraints and Guidelines

This document establishes the essential design, content, and accessibility constraints for the CraftLayers portfolio website.

\#\#\# \*\*I. Content and Deployment (MISSION)\*\*

  \* \*\*Content Update:\*\* Update all case studies and blog posts using the provided Markdown files and image mappings.  
  \* \*\*Compliance:\*\* All updates \*\*MUST\*\* strictly adhere to the designated single source of truth template, the established design system, and project guidelines.  
  \* \*\*Tone:\*\* All written content \*\*MUST\*\* be concise and sharp to maintain a professional tone.

\#\#\# \*\*II. Performance and Optimization\*\*

  \* \*\*Image Optimization:\*\* All visual assets \*\*MUST\*\* be served in modern formats (e.g., WebP) and optimized for responsiveness using \`\<picture\>\` or \`srcset\`.  
  \* \*\*Lazy Loading:\*\* Implement lazy loading for all below-the-fold images and non-critical assets.  
  \* \*\*Code Splitting:\*\* Utilize route-based or component-based code splitting to minimize the initial bundle size.

\#\#\# \*\*III. Responsive Design and Breakpoints\*\*

  \* \*\*Mobile-First Approach:\*\* All components and layouts \*\*MUST\*\* be designed and coded using a mobile-first philosophy.  
  \* \*\*Spacing:\*\* Conditionally reduce large Design System (DS) spacing tokens (xl, 2xl, 3xl) specifically on mobile viewports.  
  \* \*\*Grid System:\*\* The 12-column grid \*\*MUST\*\* collapse to a single-column or 4-column structure on mobile devices.

\#\#\# \*\*IV. Code Quality and Naming Conventions\*\*

  \* \*\*Utility Classes:\*\* Use \*\*only\*\* utility classes generated by \`tailwind.config.ts\`; custom Tailwind classes are strictly prohibited.  
  \* \*\*Component Naming:\*\* React/TSX components \*\*MUST\*\* follow PascalCase (e.g., \`ProfileHero\`).  
  \* \*\*Prop Naming:\*\* Boolean props should be prefixed with \`is-\` or \`has-\` (e.g., \`isActive\`).  
  \* \*\*CSS Variables:\*\* Direct values for color, spacing, and radius \*\*MUST\*\* reference custom properties from semantic tokens.

\#\#\# \*\*V. Error Handling and External Links\*\*

  \* \*\*404 Page:\*\* A custom 404 error page \*\*MUST\*\* be implemented, utilizing the Design System, main navigation, and a link back to the Home Page.  
  \* \*\*External Links:\*\* All external links \*\*MUST\*\* open in a new tab (\`target="\_blank"\`) and include security attributes (\`rel="noopener noreferrer"\`).
