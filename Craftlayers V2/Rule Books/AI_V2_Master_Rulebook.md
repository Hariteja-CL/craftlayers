ANTIGRAVITY V2 MASTER RULEBOOK

Purpose
Antigravity governs the transformation of raw signals into actionable decisions in AI-assisted products.

Core Pipeline
Signal → Analysis → Decision → Action

AI must convert information into operational recommendations.

---

SYSTEM RULES

SR-01
Inputs must be treated as structured signals.

SR-02
Signals must be interpreted within operational context.

SR-03
Analysis must generate ranked interventions.

SR-04
Outputs must prioritize decisions over descriptive insights.

SR-05
Every recommendation must include an execution path.

SR-06
Data must be aggregated before AI processing.

SR-07
Trigger signals must produce recommended actions.

---

UX DESIGN RULES

DR-01
Dashboards must present decision signals before raw metrics.

DR-02
AI recommendations must include rationale.

DR-03
Outputs must appear as actionable UI components.

DR-04
Action interfaces should enable one-step execution.

DR-05
Users must be able to trace the underlying data source.

DR-06
Alerts escalate through:

Signal → Context → Resolution

DR-07
Interfaces must reduce cognitive load.

---

AI GENERATION RULES

GR-01
AI must not invent organizational data.

GR-02
Responses must reference available system data.

GR-03
Avoid anthropomorphic language.

GR-04
Use operational language instead of speculation.

GR-05
Recommendations must follow intervention templates.

GR-06
If data is insufficient, request clarification.

---

SECURITY RULES

SEC-01
Sensitive data must never appear in AI responses.

SEC-02
AI analysis must operate within authorized data scopes.

SEC-03
AI must not infer personal attributes.

SEC-04
Minimum anonymity thresholds must be respected.

SEC-05
All AI actions must remain auditable.

---

OUTPUT STRUCTURE

1 Signal detected
2 Context summary
3 Strategic interpretation
4 Recommended intervention
5 Execution option

---

FAILSAFE

If safe recommendation cannot be determined:

“Insufficient data for actionable recommendation.”
